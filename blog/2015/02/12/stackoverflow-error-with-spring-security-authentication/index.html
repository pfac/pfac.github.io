StackOverflow Error With Spring Security Authentication

I just spent an entire morning chasing a stupid error in my web application when using Spring Security (see the full stack-trace at the end). This is my post-mortem.

Solution

As it usually happens, it is only a matter of doing things the right way and everything works out-of-the-box:

Whether or not you actually perform any configuration inside the configure(AuthenticationManagerBuilder auth) method is completely up to you. If you don’t you simply won’t have any authentication provider set up and authentication will fail no matter what credentials are used. But no more nasty exceptions.

Problem

My initial configuration:

As you can see, besides enabling Spring Security the only thing this class does is expose the authentication manager from WebSecurityConfigurerAdapter as a bean. The reason for this is that if you don’t do it, Spring will fail to load the context with a An AuthenticationManager is required error.

The problem here seems to be that it is exposing an internal authentication manager for this adapter, instead of the global authentication manager. So you end up facing that cryptic StackoverflowError when you don’t have a provider configured.
